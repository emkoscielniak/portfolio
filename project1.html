Project Overview

The Problem

Event hosts struggle with using multiple applications (Google Forms, Google Sheets, Partiful) to manage budgeting, collect dietary restrictions, and consider suitable caterers, creating friction and health risks when it comes to event planning.

The Solution

A unified web application that streamlines planning, collects guest data, and provides data analysis for budgeting and dietary accommodations.

As Project Lead and Lead UX Designer, I guided our team through the iterative design process to create EventGenie—a platform that reduces app fatigue and empowers event hosts to plan with confidence.

Discovery & Research

We conducted user interviews, created affinity maps, and culminated a persona and scenarios to deeply understand event hosts' pain points.

Research Methodology

  • User Interviews: Spoke with event coordinators, and club leaders
  • Affinity Mapping: Synthesized interview data into themes and patterns
  • Persona Creation: Developed a detailed user profile based on research
  • Scenario Drafting: Mapped realistic use cases and problem situations

Key Pain Points Discovered

App Fatigue

Users find the abundance of separate applications needed for event planning frustrating and overwhelming.

Coordination Anxiety

The hardest part of planning is coordinating restaurants and ensuring specific ingredients aren't contained in menu items.

Data Compilation

It is time-consuming to compile dietary restrictions from hundreds of guests into a readable format without being prone to error.

Emotional Friction

A majority of reported feelings regarding event planning were negative, including annoyance with forms and frustration coordinating vendors.

Empathy Map Insights

Our empathy mapping revealed that users felt stressed and annoyed with Google Sheets/Forms, and felt the heavy responsibility of others' food service and safety.

Design Strategy

Based on our research insights, we translated user pain points into actionable design goals and features.

User Goals

We defined three core goals based on user needs:

  1. Plan an event within a budget range
  2. Analyze budget and user trends
  3. Connect with local food providers and arrange catering

Problem Scenarios

We developed scenarios to frame the core problem, such as:

Jessica, Front Desk Attendant: Spends hours toggling between Google Suite apps to collect dietary restrictions, check budgets in spreadsheets, and search Yelp for caterers while constantly worrying about missing a critical allergy

How Might We Statements

  • HMW design app navigation that is natural and effortless for users of all ages and technical skills?
  • HMW allow users to quickly find affordable suppliers for their food?
  • HMW allow users to communicate with food suppliers more easily, especially with language barriers present?

Architecture & Prototyping

We transitioned from abstract concepts to testable prototypes through careful information architecture and user flow mapping.

Information Architecture

We established a clear core structure for the application:

  • Home → Dashboard with quick access to key features
  • Explore Restaurants → Browse and vet catering options
  • Events → Manage current and upcoming events
  • Data Analysis → Budget tracking and dietary trends
  • Profile → User settings and preferences

User Flows

We mapped detailed flows for critical tasks, such as "Plan an event within a budget range" which moves through:

  1. Sign-up / Login
  2. Create new event
  3. Set budget parameters
  4. Review suitable caterers
  5. Contact and arrange catering

Usability Testing & Validation

We tested the clickable prototype with users to validate our design decisions and identify areas for improvement.

Testing Methodology

Participants were given specific tasks to complete:

  • Create a new event
  • Review budget analysis for an event
  • Get in contact with a restaurant
  • View dietary restriction distribution data

Key Findings

Navigation Success

The main navigation bar (Home, Restaurants, Events, Data) was found to be intuitive and easy to use.

Icon Confusion

Participants had trouble finding the "Dietary Restriction Distribution" graph and "guessed where it should be."

Contact Friction

The message bubble icon was too small and blended in with the phone icon. Users gravitated toward the rightmost icon (phone).

User Preference

Users showed a preference for chat over phone contact, indicating the need to prioritize the chat feature.

Design Iterations Based on Testing

Recommendation 1: Refine Data Information Architecture

Add clear subheadings to the "Data" tab to label Budget and User Trends sections, making the dietary restriction distribution easier to find.

Recommendation 2: Improve Contact UI

Make the chat bubble larger, move it to the right side (position of preference), and clearly define it as the preferred contact method over phone.

Next Steps & Takeaways

Final Vision

EventGenie successfully reduces app fatigue and collects data for event hosts, allowing them to optimize their planning workflow and ensure accommodating, safe events for all attendees.

Impact

  • Reduced number of apps needed from 5+ to 1
  • Centralized dietary data eliminates manual compilation errors
  • Budget analysis features suggests data-driven catering decisions
  • Direct restaurant communication reduces coordination anxiety

Next Phase: High-Fidelity UI

Based on our validation testing, the next steps include:

  • Visual Design System: Develop comprehensive UI incorporating brand identity and accessibility standards
  • Budget Visualizations: Implement interactive charts and graphs for budget analysis features
  • Data Analysis Dashboard: Create clear subheadings and improved Information Architecture for the dietary restriction distribution feature
  • Responsive Design: Ensure mobile and tablet optimization for portable event management

Reflection

Leading this project taught me valuable lessons in:

  • Prioritizing Discovery: Thorough user research allowed for my team and I to deeply understand the problem we are trying to solve prior to focusing on solutions.
  • Mitigating Bias:Detailed research helped us in recognizing and addressing personal biases during the design process, a crucial action to ensure our app genuinely solved the hosts' complex planning needs.
  • Iterative process:Engaging with the full design process and seeing where artifacts fit into tho the Double Diamond model increased my confidence as a designer due to the success of creating an effective and inclusive design.
